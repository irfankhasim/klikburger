================================================================================
A. PENGENALAN — APA ITU PROJEK INI?
================================================================================

TAB KAUNTER ialah sistem Point of Sale (POS) dan pejabat belakang (Back Office)
untuk restoran burger / makanan segera. Sistem ini direka untuk mengendalikan
operasi harian restoran dalam SATU platform bersepadu:

  - Jualan di kaunter
  - Paparan pesanan dapur (kitchen display)
  - Pengurusan resit
  - Drawer tunai & syif kerja (shift management)
  - Pengiraan kos bahan mentah (FIFO — First In, First Out)
  - Menu costing (harga jual vs kos sebenar)
  - Pengurusan staf & kehadiran
  - Laporan bulanan automatik
  - Pembantu AI untuk SOP dan panduan operasi
  - MCP Server untuk integrasi AI agent (Cursor, Claude, dll.)

Projek ini dibina sebagai aplikasi web statik (HTML + CSS + JavaScript ES Modules)
dengan Google Firebase sebagai backend. TIADA server REST tersendiri (Express,
FastAPI, dll.) dalam repositori ini.


================================================================================
B. LATAR BELAKANG & MASALAH
================================================================================

Restoran kecil dan sederhana sering bergantung pada:

  1. Kalkulator manual atau POS asing yang tidak integrasi dengan stok
  2. Rekod stok yang tidak tepat (tiada kaedah FIFO)
  3. Laporan jualan dan kos yang lambat disediakan
  4. Tiada sistem terpusat untuk staf, syif kerja, dan audit
  5. Staf baru sukar dapat jawapan SOP tanpa ganggu pengurus

TAB KAUNTER menyelesaikan masalah ini dengan:

  1. POS yang TERUS menolak stok mengikut kaedah FIFO
  2. COGS (Cost of Goods Sold) dikira automatik setiap jualan
  3. Semua data disimpan dalam Cloud Firestore secara masa nyata
  4. Owner/Admin boleh pantau operasi melalui dashboard dan laporan bulanan
  5. AI assistant membantu staf jawab soalan SOP tanpa ganggu pengurus
  6. MCP Server membolehkan AI agent akses data operasi secara terkawal

G.4 POS JUALAN & CHECKOUT FIFO
--------------------------------------------------------------------------------
Modul TERAS projek.

Aliran jualan:
  Pilih menu → Tambah cart → Semak stok → Bayar → Transaction Firestore

Dalam SATU transaksi Firestore (finalizePosSaleFifo):
  1. Baca batch bahan (ingredient_batches) — susunan FIFO
  2. Tolak kuantiti stok dari batch tertua dahulu
  3. Kira COGS per barisan jualan
  4. Tulis dokumen sales
  5. Tulis pos_orders + subkoleksi items
  6. Tulis pos_receipts
  7. Tulis pos_sales_transactions
  8. Kemas kini counter (pos_meta/counters: seqOrder, seqReceipt)

FIFO (First In, First Out):
  - Setiap pembelian bahan → batch baru dalam ingredient_batches
  - Batch disusun mengikut openedAt (lot lama habis dulu)
  - COGS = kos batch sebenar yang digunakan, bukan harga purata

Fail berkaitan:
  - js/pos-order-app.js                  → UI jualan
  - js/pos-sale-fifo.js                  → Logik FIFO & COGS
  - js/pos-checkout-firestore-writer.js  → Tulis dokumen POS dalam transaksi
  - js/cost-calculator/ingredient-batch-repository.js → Urus batch
